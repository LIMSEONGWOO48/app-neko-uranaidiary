# PR4: 気分占いのメモ入力を送信後に非表示・クリア

## Summary

「気分で占う」画面で占いを送信したあと、一言メモの入力テキストが画面に残り続ける問題を修正した。  
送信後はメモ欄をクリアし、1日の占い回数上限に達した場合はメモ欄自体を非表示にする。

## 背景・課題

- 占い実行後も `TextField` に入力したメモが残る
- 回数上限到達後に画面へ戻ると、送信済みメモがそのまま表示される
- `onAppear` で `DailyLog` のメモを入力欄へ復元していたため、プライバシー面でも望ましくない

## 変更内容

### 1. 送信後にメモをクリア

- `clearMemoInput()` を追加
- 占い結果画面へ遷移する直前に `memo = ""` とキーボードフォーカス解除を実行
- 新規占い・「もう一度見る」の両方で適用

### 2. 回数上限到達後はメモ欄を非表示

- `shouldShowMemoField` を追加（`usageStatus?.canFortune` が `false` のとき非表示）
- 「今日の占い回数上限に達しました」状態で、送信済みメモが見えないようにした

### 3. 過去入力の自動復元を停止

- `loadTodayInputIfNeeded()` から `memo = todayLog.memo ?? ""` を削除
- 気分・ジャンルのみ復元。メモは `DailyLog` に保存され、過去ログでは引き続き閲覧可能

### 4. 「もう一度見る」の判定調整

- メモ欄が空の状態でも、気分・ジャンルが今日のログと一致すれば回数消費なしで再表示可能
- メモを入力している場合のみ、保存済みメモとの一致を確認
